**技术栈选择：**
- 后端：Python + Flask（保持现有架构）
- 计算引擎：Python标准库 `math` 模块
- 前端：HTML/CSS/JavaScript（保持现有架构）

**架构分层思路：**
1. **计算引擎层**：使用Python的math模块提供科学计算能力
2. **业务逻辑层**：封装科学计算函数，处理角度/弧度转换等业务规则
3. **接口层**：通过REST API暴露科学计算功能
4. **表示层**：前端界面添加科学计算按钮和显示

**关键设计决策：**
1. 使用math模块，无需额外依赖
2. 三角函数支持角度和弧度两种模式
3. 对数函数支持常用对数和自然对数
4. 幂运算支持任意指数
5. 错误处理：处理无效输入（如负数开方、0的对数等）
